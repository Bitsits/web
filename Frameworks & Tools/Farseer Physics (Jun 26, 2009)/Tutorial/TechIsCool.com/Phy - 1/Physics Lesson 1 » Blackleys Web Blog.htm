Physics Lesson 1

This will be my first tutorial on using a physics engine, this tutorial was created on request so if you have any ideas please drop by the request page and leave a comment! For this we will be using the Farseer Physics engine and it so far has been very simple and at the same time allowing for very complex shapes and special environments with differing gravity. Now our first project can use classes and functions from the other project! On to the Code! where we have all of our using Microsoft... we need to ad a few of them for starters lets add all of these

using FarseerGames.FarseerPhysics;
using FarseerGames.FarseerPhysics.Collisions;
using FarseerGames.FarseerPhysics.Controllers;
using FarseerGames.FarseerPhysics.Dynamics;

There might be a few in there we will not be using. Now we have the FarseerPhysics linked with our project! Lets keep moving Add this code just above your constructor if you don't know what it is it looks like this 'public Game1()'

PhysicsSimulator phySim;
Body ballBody;
Geom ballGeom;

Body platformOne;
Geom platformOneGeo;

Texture2D ballTex;
Texture2D platformTex;
 

The physics simulator is the backbone of the physics it will control and Handel how objects interact with each other it also will need to know what gravity is, we will talk a little more about this when we create the phy sim, Next we have a Body this handles all the effects for the object in our case it will be a ball. Then we have a Geom short for Geometry this is the outline of the object and is the main part of collisions. Then we just have 2 Texture2Ds to hold our platform and our ball if you don't know how to use spritebatch or Texture2D please read my 2D tutorials (coming soon (really soon)) Now in the LoadContent function add the following if you cant find it it looks like this 'protected override void LoadContent()'

ballTex = Content.Load<Texture2D>("Phy_ball");
platformTex = Content.Load<Texture2D>("Phy_Platform");

//Create the physics simulator
phySim = new PhysicsSimulator(new Vector2(0, 200));

//Create the ball body and geometry
ballBody = BodyFactory.Instance.CreateCircleBody(22f, 2);
ballBody.Position = new Vector2(250, 100);
ballGeom = GeomFactory.Instance.CreateCircleGeom(phySim, ballBody, 22f, 16);
 

So the first two lines are quite simple just loading our textures just for reference my ball is (64,64) and my platform is (256, 64) in pixels. The next thing is our physics simulator it can take a Vector2 as gravity or you can set it later but for now we will use a Vector2 setting gravity don't worry 200 is just an amount of force not the amount of pixels to move every second. Then we create the Body and the Geometry for the ball, since the ball is a circle and and its 64 pixels wide the radius would be 32 but since my ball does not go out to the exact pixel edge it will stay 22, when we create the body we also need to give it an amount of mass in this case we gave it 2. Then we set the position of the body to 250 , 100. Finally we need to create the geometry this is what will actually be used to check the collisions in this case we want it to be a circle we could create our own geometry and send it to the ballGeom but there is a function that will create a circle for us. In this case it is 'CreateCircleGeom' it takes the physics simulator then the body that it will be linked to, then the radius of the object, then it takes the number of edges in this ball we told it 16 this means there will be 16 piece of geometry to check the collision, the larger the number the more calculations occur but the more accurate it is.

platformOne = BodyFactory.Instance.CreateRectangleBody(256, 64, 1);
platformOne.IsStatic = true;
platformOne.Position = new Vector2(250, 300);
platformOne.Rotation = MathHelper.ToRadians(15f);
platformOneGeo = GeomFactory.Instance.CreateRectangleGeom(phySim, platformOne, 256f, 64f);
 

This goes right below the last piece of code we wrote, This uses different functions for the GeomFactory and the BodyFactory but their constructors are all understandable. The new thing we added was the rotation it takes radians and in this case it will be rotated 15 degrees clockwise.

//Add the objects to the simulator
phySim.Add(ballBody);
phySim.Add(ballGeom);

phySim.Add(platformOne);
phySim.Add(platformOneGeo);
 

One of the last things we need to do to get the simulator working is add the objects we created to the sim, I noticed that if you do not pass the body followed by the Geom sometimes it gets messed up, just make sure you pass a body and its geometry to the phySim right after each other. This might be a short time glitch with this current version. Now for the last thing simulator wise add this code in the update function now if you cant find it it looks like this 'protected override void Update(GameTime gameTime)'

//Update The Physics Sim
phySim.Update(gameTime.ElapsedGameTime.Milliseconds * .001f);
 

This just updates the physics engine. If you try to run the game right now it will work and the simulation will run the only problem is we cant see anything! So add this code to your draw function again if you cant find it it looks like 'protected override void Draw(GameTime gameTime)'

spriteBatch.Begin();
spriteBatch.Draw(ballTex, ballBody.Position, null, Color.White, ballBody.Rotation, new Vector2(32, 32), 1f, SpriteEffects.None, 1f);
spriteBatch.Draw(platformTex, platformOne.Position, null, Color.White, platformOne.Rotation, new Vector2(128, 32), 1f, SpriteEffects.None, 1f);
spriteBatch.End();
 

So there are a few things to note here, first the Vector2 Origin should be the center of the object this is what the engine expects it to be if it is not some strange things will occur. The other thing is that we use the position of the body! If all this came together properly you should be able to run and have your ball fall hit the platform all roll off. and it should look something like this.

2D PhyLesson 1

Lesson Code: (without keyboard input and ball reset)

using System;
using System.Collections.Generic;
using System.Linq;
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Audio;
using Microsoft.Xna.Framework.Content;
using Microsoft.Xna.Framework.GamerServices;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;
using Microsoft.Xna.Framework.Media;
using Microsoft.Xna.Framework.Net;
using Microsoft.Xna.Framework.Storage;

using FarseerGames.FarseerPhysics;
using FarseerGames.FarseerPhysics.Collisions;
using FarseerGames.FarseerPhysics.Controllers;
using FarseerGames.FarseerPhysics.Dynamics;
using FarseerGames.FarseerPhysics.Factories;
using FarseerGames.FarseerPhysics.Interfaces;
using FarseerGames.FarseerPhysics.Mathematics;


namespace _2D_Phy_Lesson1
{
    /// <summary>
    /// This is the main type for your game
    /// </summary>
    public class Game1 : Microsoft.Xna.Framework.Game
    {
        GraphicsDeviceManager graphics;
        SpriteBatch spriteBatch;

        PhysicsSimulator phySim;
        Body ballBody;
        Geom ballGeom;

        Body platformOne;
        Geom platformOneGeo;

        Texture2D ballTex;
        Texture2D platformTex;

        public Game1()
        {
            graphics = new GraphicsDeviceManager(this);
            Content.RootDirectory = "Content";
        }

        /// <summary>
        /// Allows the game to perform any initialization it needs to before starting to run.
        /// This is where it can query for any required services and load any non-graphic
        /// related content.  Calling base.Initialize will enumerate through any components
        /// and initialize them as well.
        /// </summary>
        protected override void Initialize()
        {
            // TODO: Add your initialization logic here

            base.Initialize();
        }

        /// <summary>
        /// LoadContent will be called once per game and is the place to load
        /// all of your content.
        /// </summary>
        protected override void LoadContent()
        {
            // Create a new SpriteBatch, which can be used to draw textures.
            spriteBatch = new SpriteBatch(GraphicsDevice);

            ballTex = Content.Load<Texture2D>("Phy_ball");
            platformTex = Content.Load<Texture2D>("Phy_Platform");

            //Create the physics simulator
            phySim = new PhysicsSimulator(new Vector2(0, 200));

            //Create the ball body and geometry
            ballBody = BodyFactory.Instance.CreateCircleBody(22f, 2);
            ballBody.Position = new Vector2(250, 100);
            ballGeom = GeomFactory.Instance.CreateCircleGeom(phySim, ballBody, 22f, 16);

            //Create the platform body and geometry
            platformOne = BodyFactory.Instance.CreateRectangleBody(256, 64, 1);
            platformOne.IsStatic = true;
            platformOne.Position = new Vector2(250, 300);
            platformOne.Rotation = MathHelper.ToRadians(15f);
            platformOneGeo = GeomFactory.Instance.CreateRectangleGeom(phySim, platformOne, 256f, 64f);

            //Add the objects to the simulator
            phySim.Add(ballBody);
            phySim.Add(ballGeom);

            phySim.Add(platformOne);
            phySim.Add(platformOneGeo);
        }

        /// <summary>
        /// UnloadContent will be called once per game and is the place to unload
        /// all content.
        /// </summary>
        protected override void UnloadContent()
        {
            // TODO: Unload any non ContentManager content here
        }

        /// <summary>
        /// Allows the game to run logic such as updating the world,
        /// checking for collisions, gathering input, and playing audio.
        /// </summary>
        /// <param name="gameTime">Provides a snapshot of timing values.</param>
        protected override void Update(GameTime gameTime)
        {
            // Allows the game to exit
            if (GamePad.GetState(PlayerIndex.One).Buttons.Back == ButtonState.Pressed)
                this.Exit();

            //Update The Physics Sim
            phySim.Update(gameTime.ElapsedGameTime.Milliseconds * .001f);

            base.Update(gameTime);
        }

        /// <summary>
        /// This is called when the game should draw itself.
        /// </summary>
        /// <param name="gameTime">Provides a snapshot of timing values.</param>
        protected override void Draw(GameTime gameTime)
        {
            GraphicsDevice.Clear(Color.CornflowerBlue);

            spriteBatch.Begin();
            spriteBatch.Draw(ballTex, ballBody.Position, null, Color.White, ballBody.Rotation, new Vector2(32, 32), 1f, SpriteEffects.None, 1f);
            spriteBatch.Draw(platformTex, platformOne.Position, null, Color.White, platformOne.Rotation, new Vector2(128, 32), 1f, SpriteEffects.None, 1f);
            spriteBatch.End();

            base.Draw(gameTime);
        }
    }
}
